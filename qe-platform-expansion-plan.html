4. Core Quality Guidelines

Research synthesizing Microsoft Copilot Best Practices and Playwright docs establishes these governing rules:

  1. Adopt the Plan-Generate-Heal Loop: Utilize Copilot as a generation engine and Playwright Healer agents as maintenance safety nets. (Current: Missing)
  2. Web-First Semantic Resilience: Prefer intentional locators piercing shadow DOMs (getByRole, getByText) strictly avoiding XPath/CSS fragility. (Current: Partial)
  3. Shift-Left with Non-Functional Checks: Test automation must integrate WCAG compliance (A11y) inside standard functional flows. (Current: Missing)
  4. Scale Horizontally, Not Vertically: Rely on test-level fullyParallel configuration combined with CI container --shard configurations to manage growth. (Current: Missing)
  5. Zero Hardcoded Interventions: Eliminate waitForTimeout and utilize the native web-first auto-wait paired with Trace Viewer visualization for race-conditions. (Current: Covered)
