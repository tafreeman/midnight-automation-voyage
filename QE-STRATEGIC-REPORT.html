Executive Summary

Consolidated findings from 8 independent audit and research documents, cross-referenced against Playwright official documentation, ISTQB certification syllabi, the TMMi Foundation model, and the 2025-26 World Quality Report.

Core Diagnosis (Unanimous Across All Reports)

Strong content, weak platform. The instructional material is production-quality — 15 well-structured modules, dual-audience design, the CARD prompting framework, and HITL governance. But the delivery mechanism (no persistence, no assessment, no accessibility) and curriculum coverage (no visual testing, no auth fixtures, no mobile, GitLab-only CI) undermine enterprise credibility. The platform operates at TMMi Level 2 (Managed/Repeatable). The recommended path targets Level 4 (Measured) within 6 months.

Three Findings That Define the Strategy

1. Assessment Layer is Structurally Absent

Only 2 of 15 modules (13%) include exercises. Prompt templates appear in 1 module (7%). The platform teaches test writing but does not require learners to write. This is the single most critical gap for enterprise adoption, where measurable training outcomes are non-negotiable.

2. Playwright Coverage Stops at Functional

The 33 reference tests demonstrate strong foundational patterns, but zero coverage of auth fixtures, visual regression, accessibility scanning, mobile emulation, cross-browser config, network mocking, or trace debugging. Estimated Playwright API surface coverage: ~30%.

3. Platform UX Undermines Content Quality

13 UI/UX issues identified, including monospace body text (10-15% reading speed reduction), no progress persistence (state lost on refresh), no keyboard navigation, and a sidebar that fully unmounts when closed. Total fix effort estimated at ~9 hours — high ROI.

Section 03

Platform Strengths

Assets identified unanimously across all audits. These should be preserved and amplified in any expansion.

Manual-to-Automated Decomposition

The test-case-mapping.md document walks through a 26-step manual test, explains why it fails as a single automated test, decomposes it into 7 focused tests, and provides CARD prompts for each. Aligns with ISTQB CTAL-TAE test design techniques.

Evidence: test-cases/test-case-mapping.md

CARD Prompting Formula

Context / Actions / Rules / Data structured prompting is an original contribution. No competing platform teaches formalized prompt engineering for test generation. WQR 2025-26 ranks GenAI as #1 QE skill (63%).

Evidence: Module 06 (AI Skills)

Dual-Track Audience Design

Every module tagged with audience indicators. Non-coder path is first-class (Modules 07, 14). Mirrors UiPath Academy multi-path patterns but integrates both tracks in a single curriculum.

Evidence: Audience tags across Modules 01-15

HITL Governance Framework

10-point Human-in-the-Loop review checklist covering assertion presence, selector strategy, data independence, and wait anti-patterns. Aligns with ISTQB CT-TAS governance and Tricentis quality gate requirements.

Evidence: Module 13 (Governance)

Selector Convention Enforcement

100% data-testid usage across all 33 reference tests. Zero waitForTimeout anti-patterns. One-test-per-behavior discipline. Comment prefixes for traceability (L1, S2, etc.).

Evidence: test-cases/examples/*.spec.ts

Purpose-Built Test Target

Practice app implements realistic patterns: authentication with lockout, multi-step wizard with data preservation, paginated data table with sort/filter. CheckoutProvider context and route guards demonstrate real-world SPA architecture.

Evidence: practice-app/src/App.tsx

DiffCodeBlock Exercise UX

Starter-to-solution inline comparison with progressive hint disclosure and copy-to-clipboard. Called "the strongest UX pattern in the app" by the design audit.

Evidence: DESIGN-AUDIT.md

Key Insight: Graceful Degradation

The three-phase structure is designed so that each phase delivers standalone value. If the project stops after Phase 2, the platform is still enterprise-credible at TMMi Level 3 with 21 modules, 50+ tests, visual/a11y/auth coverage, and a formal assessment system. Phase 3 is additive, not foundational.

Section 09

Architecture Decision Records

10 ADR candidates with sufficient backing from multiple audit documents to draft immediately. Ordered by implementation wave.

ADR-001: Adopt Inter as Primary Body Font

JetBrains Mono applied to all UI text via inline style, reducing reading speed 10-15% (Arditi & Cho, 2005). Inter is loaded but overridden.

Remove inline fontFamily from App.tsx. Use Inter for body, JetBrains Mono for code and headings only.

5 minutes. High impact

ADR-002: LocalStorage Progress Persistence

All completion state lost on refresh. Dealbreaker for multi-session learning across a 15-module course.

Persist completedLessons and currentLesson to localStorage with a versioned key. No server dependency.

30 minutes. High impact

ADR-003: Decouple Navigation from Completion

goNext() calls markComplete() on every navigation, inflating completion metrics. Violates Kirkpatrick Level 2.

Remove markComplete from goNext(). Retain explicit "Mark Complete" button. Optionally gate behind quiz attempt.

10 minutes. High impact

ADR-004: Auth Fixtures & StorageState Module

No reusable auth pattern. Auth fixtures are the #1 Playwright anti-pattern gap cited across all audits.

Module 16: storageState, global-setup, per-role fixtures. Admin Panel practice page for multi-role scenarios.

Module + feature. P1 Gap

ADR-005: Accessibility Testing Curriculum

No WCAG-aware automation taught. @axe-core/playwright is a 5-line addition. 57% of issues auto-detectable.

Module 18: axe-core integration + violation interpretation. Settings page with 3 intentional a11y violations.

Module + feature. P1 Gap

ADR-006: Visual Regression Testing Module

First-party toHaveScreenshot() requires no additional dependencies. CSS/layout regression undetectable today.

Module 17: screenshot comparison, masking, baselines, CI integration. Dashboard enhancement for test targets.

Module + feature. P1 Gap

ADR-007: Three-Tier Certification System

13% exercise coverage, 67% quiz coverage. Cannot demonstrate learning outcomes to enterprise stakeholders.

Bronze / Silver / Gold tiers. Quiz attempt required before completion. Capstone rubric aligned with HITL checklist.

Backfill + module + UI. P1 Gap

ADR-008: URL Routing for Deep Linking

No URL routing means modules cannot be bookmarked, shared, or linked from external docs. Critical for team rollouts.

Hash-based routing (#lesson/9) or React Router. Root URL loads Module 1.

1-2 hours. Medium impact

ADR-009: Multi-Platform CI/CD Coverage

Only GitLab CI covered. GitHub Actions is the dominant CI platform; Azure DevOps is standard in Microsoft-stack enterprises.

Module 27 (GitHub Actions) with workflow YAML, artifact upload, PR checks, matrix strategy. Compare against GitLab CI.

Module. P2 Gap

ADR-010: Collapsed Sidebar Rail Pattern

Closing the sidebar removes it entirely (return null). Layout shift, loss of navigation context.

Replace with 40px collapsed rail containing toggle icon. Persistent re-open target.

30 minutes. High impact
