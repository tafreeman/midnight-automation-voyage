## Slide: Human in the Loop

**Thesis:** Accelerated by AI. Governed by human expertise.

**Three proof points:**

1. **Governance First** — Approvals and legal clearance came before acceleration. Risk, legal, and compliance teams approved AI use before production release.

2. **Quality Gates** — Every AI-assisted line was reviewed before merge. More time was invested in review discipline, not less. Unit and integration testing validated output. No merge moved forward until all gates passed.

3. **Context Engineering** — Standardized prompts, process learning sessions, and developer expertise provided the system full context. Architecture docs, data models, and coding standards were embedded in prompts. Variance dropped because outputs stayed anchored to the real system.

**Results:**
- 50% reduction in development time
- 90% of production code developed via AI
- 2 months from idea to production
- 0 critical defects post-deployment

**Callout:** "AI generated code. Humans owned the decisions."

## Slide: Hurdles We Overcame

| Hurdle | Challenge | Fix |
|--------|-----------|-----|
| Prompt Standardization | Ad-hoc prompts created uneven quality and rework | Versioned prompt templates added consistent architecture context |
| Process Realignment | Traditional reviews not built for AI output | AI-specific review checklists made quality gates explicit |
| Governance Clearance | Policy approval required before production use | Risk and legal teams engaged early to set guardrails |
| Team Enablement | Skill levels with AI tools were uneven | Hands-on hackathon built shared confidence before delivery |

**Callout:** "The early friction became the guardrails that let the team move fast later."
