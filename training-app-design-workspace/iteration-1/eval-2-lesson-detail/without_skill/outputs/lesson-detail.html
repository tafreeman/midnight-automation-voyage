Writing Your First Playwright Test

Navigate to pages and interact with elements using page.goto() and page.click()

All Roles Hands-on ~15 min

What You Will Learn

Every Playwright test starts with two fundamental actions: navigating to a URL and interacting with elements on the page. In this lesson you will learn page.goto() for navigation and page.click() for triggering user interactions. By the end, you will write a working test against the practice app's login page.

These two methods form the foundation of every Playwright test. Master them and you can automate any user workflow.

Navigating with page.goto()

page.goto(url) navigates the browser to the given URL. Playwright automatically waits for the page to reach the load state before continuing. If you configure a baseURL in your Playwright config, you can use relative paths.

typescript 
// Navigate to an absolute URL
await page.goto('http://localhost:5173/login');

// With baseURL configured, use a relative path
await page.goto('/login');

// Wait for a specific load state
await page.goto('/dashboard', {
  waitUntil: 'networkidle'
});

Tip: Always set baseURL in playwright.config.ts so your tests use relative paths. This makes them portable across environments.

goto() Options Reference

Option Default Description
waitUntil 'load' When to consider navigation succeeded: 'load', 'domcontentloaded', 'networkidle', 'commit'
timeout 30000 Max time in ms to wait for navigation
referer undefined Set referer header value

Clicking Elements with page.click()

page.click(selector) finds an element matching the selector, scrolls it into view, and clicks it. Playwright automatically waits for the element to be visible, enabled, and stable before clicking. The preferred selectors are data-testid attributes and role-based locators.

typescript 
// Click by data-testid (recommended)
await page.click('[data-testid="login-button"]');

// Modern locator API (preferred)
await page.getByTestId('login-button').click();

// Click by role (great for accessibility)
await page.getByRole('button', { name: 'Log In' }).click();

// Click by visible text
await page.getByText('Submit').click();

Warning: Avoid CSS selectors like .btn-primary or #root > div > button. These break when the UI is refactored. Always prefer data-testid or role-based locators.

Putting It Together: A Complete Login Test

Combining page.goto(), page.fill(), and page.click() lets you automate a complete login flow. Notice the Arrange / Act / Assert structure.

typescript Complete login test 
import { test, expect } from '@playwright/test';

test('successful login redirects to dashboard', async ({ page }) => {
  // ARRANGE - navigate to the login page
  await page.goto('/login');

  // ACT - fill in credentials and submit
  await page.getByTestId('email-input').fill('user@test.com');
  await page.getByTestId('password-input').fill('Password123!');
  await page.getByTestId('login-button').click();

  // ASSERT - verify we landed on the dashboard
  await expect(page).toHaveURL(/dashboard/);
});

Tip: Notice the Arrange / Act / Assert pattern. Every well-structured test follows this. When reviewing tests, check that each section is present.
